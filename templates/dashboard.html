Status Codes Distribution
A status codes graph displays the distribution of HTTP status codes versus their frequency in a dataset or system. HTTP status codes, ranging from 100 to 599, signify the outcome of client-server interactions, such as success (2xx), redirection (3xx), client errors (4xx), or server errors (5xx). Each status code is represented by a bar or point on the graph, with the height or position indicating its frequency. This visualization allows users to quickly identify common status codes and potential issues, aiding in the monitoring and optimization of web services.
Traffic Label Distribution
Traffic Label Distribution refers to the distribution or breakdown of different types of network traffic within a network. It provides insights into the composition and characteristics of the traffic traversing a network infrastructure. These labels can include various categories such as web browsing, file transfers, video streaming, VoIP (Voice over Internet Protocol), gaming, and more. Analyzing the traffic label distribution helps network administrators understand the nature of the network usage, identify potential bottlenecks or security threats, optimize network resources, and tailor network policies or configurations to better accommodate the predominant types of traffic.
Requests Over Time
A "requests over time" graph is a visualization that illustrates the trend of requests or interactions made to a system or service over a specified period. Typically, the x-axis represents time, which could be divided into discrete intervals such as hours, days, or months, depending on the timeframe of interest. The y-axis typically represents the count or frequency of requests. Each data point on the graph represents the number of requests recorded during a specific time interval. By observing the graph, one can discern patterns, spikes, or trends in the volume of requests over time. This information is valuable for monitoring system usage, identifying peak usage periods, detecting anomalies, and planning resource allocation or scaling strategies to ensure the system can handle varying levels of demand effectively.
Size of Requests by Traffic Label

The Size of Requests by Traffic Label" refers to a visualization that illustrates the distribution of request sizes across different traffic labels or categories within a network. In this graph, each traffic label (e.g., web browsing, video streaming, file transfers) represents a distinct category of network activity. The x-axis typically represents the traffic labels, while the y-axis represents the size of the requests, often measured in bytes or another appropriate unit. Each bar or data point on the graph represents the average or aggregated size of requests associated with a specific traffic label. Analyzing this graph provides insights into the data consumption patterns of different types of network activities. It helps in understanding the resource requirements, optimizing network bandwidth allocation, and identifying potential performance bottlenecks or anomalies across various traffic categories.

Method Distribution
A Method Distribution graph displays the frequency of HTTP request methods like GET, POST, PUT, and DELETE. It visualizes the distribution of these methods within a dataset or system, aiding in understanding usage patterns and optimizing system configurations.
Status Distribution
A Status Distribution graph visualizes the frequency of HTTP status codes (e.g., 200, 404, 500) within a dataset or system. It illustrates the distribution of success, error, and redirection responses, aiding in monitoring system health, identifying issues, and optimizing performance.
User Agent Distribution
A User Agent Distribution graph showcases the frequency of different user agent strings accessing a system or website. It helps identify the browsers, devices, and applications used by visitors, aiding in website optimization, compatibility testing, and targeted marketing strategies.
